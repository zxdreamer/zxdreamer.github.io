fork引发的死锁问题

发表于 2019-09-03

fork用于创建子进程，当父进程中拥有了子线程，fork出的子进程只会拥有执行fork函数的线程的副本，并不会拥有全部子线程
也就是说此时子进程其实是”孤家寡人”，但是子进程会用于父进程创建的互斥锁，条件变量和读写锁的副本，同时会保留当时的状态。对于一个互斥锁。如果子线程中获得
了锁，执行fork的线程阻塞，此时如果fork出的子进程再加锁，此时在子进程中就出现了死锁。

执行结果：
—sub thread lock—
—sub thread unlock—
—father lock—
—father unlocked—
—sub thread lock—
我们发现子进程并没有执行。
